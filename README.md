# ERP 系统

基于 [go-zero](https://github.com/zeromicro/go-zero) 的微服务 ERP 管理系统，涵盖进销存、人力资源、财务、生产管理等核心业务模块。

## 技术栈

| 层级 | 技术选型 |
|------|---------|
| 微服务框架 | go-zero (REST + gRPC) |
| 服务注册发现 | Etcd |
| 数据库 | MySQL 8.0 (按域分库) |
| 缓存 | Redis 7 |
| 分布式事务 | DTM (Saga 模式) |
| 对象存储 | Cloudflare R2 (S3 兼容) |
| 网关 | Nginx (HTTPS + CORS) |
| ID 生成 | Sonyflake 雪花算法 |
| 认证鉴权 | JWT + RBAC 动态权限 |

## 项目结构

```

│   │   └── rpc/                  # gRPC     (端口 9001)

│   ├── hr/                       # 人力资源管理 (8003 / 9003)

│   ├── customer/                 # 客户管理  (8007 / 9007)
│   ├── sale/                     # 销售管理  (8008 / 9008)
```

每个服务模块独立包含 `api/`（REST 接口）和 `rpc/`（gRPC 服务）两个子目录，各自拥有独立的配置文件、处理器、逻辑层和数据访问层。

## 服务模块

### 认证与权限 (auth)

系统的核心安全模块，提供完整的 RBAC 权限控制。

- **API 接口**: 角色管理、权限管理、用户-角色绑定、角色-权限绑定
- **RPC 服务**: 权限树查询、角色增删改查、用户角色分配
- **鉴权机制**: 所有其他服务的 API 请求均通过 `AuthMiddleware` 校验 JWT Token，并从数据库动态加载 URL + Method 级别的权限控制

### 用户管理 (user)

- **API 接口**: 登录注册、密码找回、Token 刷新、个人信息管理、头像上传
- **RPC 服务**: 用户 CRUD、批量导入、按工号查询、登录验证
- **特性**: 验证码发送、JWT 黑名单（Redis 存储）、Token 版本控制强制下线

### 人力资源管理 (hr)

系统中最复杂的模块之一，覆盖员工全生命周期管理。

- **组织架构**: 部门管理（树形结构）、岗位管理（含批量分配）
- **员工管理**: 入职、调薪、批量导入（Excel）、按工号查询
- **考勤管理**: 打卡、补卡申请与审批、迟到/早退/缺勤判定
- **请假管理**: 请假申请、审批流程
- **离职管理**: 离职申请、审批（集成 DTM 分布式事务保证数据一致性）
- **薪酬管理**: 月度薪酬生成（Cron 定时任务）、审批、提交财务付款
- **RPC 服务**: 考勤记录、补卡申请、部门树、员工详情、请假单、薪酬记录、岗位员工列表

### 商品管理 (product)

- **API 接口**: 商品列表、分类树、批次管理
- **RPC 服务**: 商品 CRUD、产品分类（树形结构）、批次管理
- **特性**: 区分原材料与成品（`isMaterial` 标志），为生产管理和库存管理提供基础数据

### 库存管理 (inventory)

- **API 接口**: 仓库管理、库存查询、库存调整、库存锁定、交易记录、低库存预警
- **RPC 服务**: 库存 CRUD、仓库管理、库存交易记录
- **特性**: 跟踪实时库存、安全库存、锁定库存（预留订单），所有出入库操作记录交易日志并关联业务类型（采购/销售/生产/盘盈/盘亏）

### 供应商管理 (supplier)

- **API 接口**: 供应商搜索与 CRUD、供应商评估
- **RPC 服务**: 供应商信息、评估记录
- **特性**: 信用等级（A-E）、质量/交付/服务多维度评分

### 客户管理 (customer)

- **API 接口**: 客户搜索与 CRUD、客户分类、满意度调查
- **RPC 服务**: 客户信息、分类管理、满意度调查
- **特性**: 信用额度、账期管理、质量/交付/服务满意度评分

### 销售管理 (sale)

- **API 接口**: 销售订单（含明细上传）、销售出库
- **RPC 服务**: 销售订单 CRUD、销售出库单
- **业务流程**: 销售订单 → 确认 → 部分发货 → 完成 / 取消
- **库存联动**: 出库操作自动扣减对应商品的库存

### 采购管理 (purchase)

- **API 接口**: 采购申请、采购订单、采购入库（支持从申请/订单创建）
- **RPC 服务**: 采购申请单、采购订单、采购入库单
- **业务流程**: 采购申请 → 审批 → 采购订单（可由申请单生成或直接创建） → 采购入库单（可按未出库数量生成）
- **库存联动**: 入库操作自动增加库存
- **外部依赖**: 供应商信息、商品信息

### 财务管理 (finance)

- **API 接口**: 固定资产管理、付款记录、收款记录、工资支付
- **RPC 服务**: 固定资产、付款/收款记录、工资支付
- **特性**:
  - 固定资产折旧管理
  - 付款记录关联采购订单
  - 收款记录关联销售订单
  - 工资支付与 HR 薪酬模块对接（通过 DTM Saga 事务保证跨服务数据一致性）

### 文件存储 (image)

- **API 接口**: 图片上传、搜索、删除
- **RPC 服务**: 图片元数据管理
- **特性**: 接入 Cloudflare R2 对象存储，使用 SHA256 哈希去重，支持商品图片、考勤附件、离职/请假文档等多种业务类型

### 生产管理 (production)

- **API 接口**: BOM（物料清单）管理、生产工单、领料申请
- **RPC 服务**: BOM CRUD、BOM 明细、工单管理、领料申请审批
- **特性**:
  - BOM 支持损耗率配置
  - 工单状态流转：未开始 → 已领料 → 生产中 → 已完成 → 已入库
  - 领料申请审批流程

## 公共模块 (common)

| 包 | 说明 |
|----|------|
| `jwt/` | JWT Token 生成与解析，Claims 包含用户 ID、工号、角色 ID 列表等 |
| `xcode/` | 统一错误码定义（HTTP/gRPC 双向映射） |
| `interceptors/` | gRPC 客户端/服务端错误拦截器 |
| `util/` | 雪花 ID 生成、业务单据号生成、上下文工具 |
| `xtypes/` | 权限常量定义（按业务域分组） |
| `upload/` | 文件上传抽象层（R2 / 本地） |
| `encrypt/` | SHA256 文件哈希、通用加密工具 |
| `xtime/` | 时间工具（考勤判定、标准上下班时间） |
| `excel/` | Excel 导入导出（基于 excelize） |
| `response/` | 统一成功响应封装 |

### 通信方式

- **API ↔ RPC**: 同一服务内的 API 层通过 gRPC 调用对应 RPC 服务
- **RPC ↔ RPC**: 跨服务调用（如 hr-rpc 调用 user-rpc，finance-rpc 调用 hr-rpc）
- **服务发现**: 所有 RPC 服务注册到 Etcd，客户端通过 Etcd 动态发现服务实例
- **分布式事务**: 涉及多服务的写操作（如入职流程、离职审批、薪酬发放）通过 DTM 的 Msg 模式保证最终一致性
